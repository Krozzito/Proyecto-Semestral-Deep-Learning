\column{0.55}
\block{Resumen Comparativo}{
\large
Presentamos una \textbf{comparativa empírica} de tres arquitecturas deep learning para clasificar galaxias \textbf{clumpy} (morfología grumosa, formación estelar intensa) en imágenes \textbf{JWST}. Evaluamos: (1) \textbf{ConvNeXt-Tiny} solo imagen, (2) \textbf{ConvNeXt-Tiny + MLP} híbrido con 15 morfometrías estadísticas, y (3) \textbf{ResNet-50 + MLP} híbrido. Todos entrenados con \textbf{Focal Loss} para manejar desbalance severo (9.8\% clumpy), validados con \textbf{Stratified CV} (5 o 10 folds) y desplegados como \textbf{ensemble}. El modelo \textbf{ConvNeXt híbrido v2.1} alcanza el mejor F1 = \textbf{0.62 $\pm$ 0.12} y ROC-AUC = \textbf{0.89}, superando a ResNet-50 (F1=0.50) y al baseline solo-imagen (F1=0.58). El pipeline fue auditado para detectar y corregir \textit{data leakage} en versiones previas.
}

% -------- COLUMNA 1: INTRODUCCIÓN + DATASET --------
\column{0.25}
\block{\seccion{1}{Introducción}}{
\large
Las \textbf{galaxias clumpy} presentan múltiples núcleos brillantes, indicadores de \textbf{formación estelar intensa} en el universo temprano. Su clasificación automática es clave para entender la evolución galáctica, pero requiere superar el desbalance severo (solo 9.8\% son clumpy).

\vspace{2mm}
\textbf{Dataset:}
\begin{itemize}\itemsep1pt
\item \textbf{1442 galaxias} JWST/NIRCam (PNG RGB 240$\times$240 px)
\item \textbf{Clumpy:} \textcolor{clumpyred}{141 (9.8\%)}
\item \textbf{No clumpy:} \textcolor{noclumpygray}{1301 (90.2\%)}
\item \textbf{15 morfometrías} (statmorph): Gini, $M_{20}$, $F(G, M_{20})$, $S(G, M_{20})$, concentración, asimetría, suavidad, Sérsic $n$, $r_{\text{half}}$, ellipticidad, elongación, $r_{\text{petro,circ}}$, $r_{20}$, $r_{80}$
\end{itemize}

\vspace{2mm}
\textbf{Objetivo:} Comparar arquitecturas solo-imagen vs híbridas (imagen + tabular) para determinar si las morfometrías aportan valor sobre representaciones convolucionales densas.
}

% -------- COLUMNA 1: DISCUSIÓN --------
\column{0.5}
\block{\seccion{8}{Discusión Comparativa}}{
\large
\textbf{¿Por qué ConvNeXt híbrido v2.1 es el mejor?}
\begin{itemize}\itemsep1pt
\item \textbf{Fusión tardía} (imagen + tabular) permite que el backbone capture features visuales densos mientras el MLP aporta información morfométrica explícita.
\item \textbf{15 features tabulares} (vs 10 en ResNet) incluye $F(G, M_{20})$, $S(G, M_{20})$, radios de Petrosian, que capturan estructura espacial complementaria.
\item \textbf{10-Fold CV + TTA} reduce varianza en evaluación (std F1 = 0.12 vs 0.07 en 5-fold).
\end{itemize}

\vspace{2mm}
\textbf{¿Por qué ResNet-50 falla?}
\begin{itemize}\itemsep1pt
\item \textbf{Overparameterización:} 2048 features de imagen + 128 tabulares = 2176 dimensions para solo 141 clumpy. Overfitting severo.
\item \textbf{Menos datos de entrenamiento} (5-fold vs 10-fold) y sin TTA.
\item \textbf{Arquitectura más antigua:} ResNet-50 (2015) vs ConvNeXt (2022).
\end{itemize}

\vspace{2mm}
\textbf{¿Vale la pena el componente híbrido?}
\begin{itemize}\itemsep1pt
\item \textbf{Sí, marginalmente:} +4\% F1 sobre solo-imagen (0.62 vs 0.58).
\item \textbf{Pero:} ConvNeXt ya captura buena parte de las morfometrías implícitamente. La ganancia híbrida es modesta.
\item \textbf{Tradeoff:} Complejidad adicional (scaler por fold, 15 features) vs ganancia pequeña.
\end{itemize}

\vspace{2mm}
\textbf{Auditoría de bugs:} Se detectó y corrigió \textit{data leakage} en versiones previas (unpacking incorrecto de \texttt{make\_loaders} que evaluaba sobre train con oversampling). Esto inflaba F1 a 0.76. El F1 real depurado es 0.62.
}

% -------- COLUMNA 2: CONCLUSIONES + TRABAJO FUTURO --------
\column{0.5}
\block{\seccion{9}{Conclusiones y Trabajo Futuro}}{
\large
\textbf{Conclusiones:}
\begin{itemize}\itemsep1pt
\item \textbf{ConvNeXt híbrido v2.1} es el modelo recomendado (F1=0.62, ROC-AUC=0.89), superando a ResNet-50 (F1=0.50) y al baseline solo-imagen (F1=0.58).
\item El \textbf{componente híbrido} (imagen + tabular) aporta +4\% F1, pero la ganancia es marginal: ConvNeXt ya captura morfometrías implícitamente.
\item \textbf{ResNet-50} bajo rendimiento por overparameterización y arquitectura más antigua.
\item La \textbf{auditoría de bugs} fue crítica: el pipeline inicial reportaba F1=0.76 (inflado por leakage). El F1 real es 0.62.
\end{itemize}

\vspace{2mm}
\textbf{Direcciones futuras:}
\begin{itemize}\itemsep1pt
\item \textbf{Pre-entrenamiento self-supervised} (DINO, MAE) sobre imágenes JWST no etiquetadas.
\item \textbf{Ensemble multi-arquitectura}: combinar ConvNeXt + EfficientNet + Vision Transformer.
\item \textbf{Segmentación de clumps}: pasar de clasificación binaria a localización de clumps individuales.
\item \textbf{Anotación adicional}: expandir el dataset con más expertos y validación espectroscópica.
\end{itemize}
}
